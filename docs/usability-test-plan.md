# Usability Test Plan – Contributor Apply Flow

## Objective
Identify friction points in the end-to-end contributor apply flow:
**Connect wallet → Find an issue → Apply → Verify application**

## Participants
- **Target:** 5 participants unfamiliar with Stellar / blockchain wallets
- **Recruitment channels:** Discord #general, Twitter/X DM, Telegram community
- **Screener:** No prior Stellar wallet experience; GitHub account required

## Task Scenario (read to participant)
> "You've heard about AlignmentDrips and want to contribute to an open-source project.
> Starting from the homepage, connect your wallet, find an open issue that interests you,
> apply for it, and then confirm your application was submitted successfully."

## Session Structure (30 min each)
| # | Activity | Time |
|---|---|---|
| 1 | Welcome + consent form | 5 min |
| 2 | Background questions (Stellar/crypto familiarity) | 5 min |
| 3 | Task walkthrough (think-aloud) | 15 min |
| 4 | Debrief + satisfaction rating (1–5) | 5 min |

## Recording & Consent
- Sessions recorded via Loom / Google Meet (audio + screen)
- Consent form must be signed before recording starts (see `consent-form.md`)
- Data stored in private drive; deleted 90 days after analysis

## Metrics Collected
- Task completion rate
- Time-on-task
- Number of errors / dead-ends
- Satisfaction score (post-task)
- Verbatim quotes

## Analysis Method
After all 5 sessions: affinity mapping of observations → rank by frequency + severity → top 3 findings → one follow-up GitHub issue per finding.
