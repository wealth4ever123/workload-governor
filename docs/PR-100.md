# PR: Responsive Event History Table

**Branch:** `feature/100-event-history-table`  
**Closes:** #100

## Summary

Adds `src/EventHistoryTable.js` — a zero-dependency vanilla JS ES module that renders
the contract event history as a fully responsive, accessible data table.

## Changes

- `src/EventHistoryTable.js` — new component

## Behaviour

| Viewport | Layout |
|---|---|
| > 640 px | 5-column `<table>`: Date, Action, Org, Issue, Status |
| ≤ 640 px | Card list showing Date, Action, Status |

- **Sortable columns:** Date and Action — client-side, no extra API calls.
- **Row / card click** toggles an expanded detail section showing ledger hash and full addresses (actor, contributor).
- Keyboard accessible (Enter / Space to sort or expand); `aria-sort` and `aria-expanded` attributes set correctly.

## Acceptance Criteria

- [x] Table renders correctly on desktop and mobile
- [x] Sort works client-side without additional API calls
- [x] Expanded row shows all available event data (ledger hash, actor address, contributor address)
