# PR: 404 and Error Page UI

**Branch:** `feature/101-error-pages`  
**Closes:** #101

## Summary

Adds branded, server-renderable 404 and generic error pages with the platform's top nav.

## Changes

| File | Description |
|---|---|
| `src/pages/404.html` | 404 page – illustration, message, dashboard link |
| `src/pages/error.html` | 500 / generic error page – error code, message, retry button |
| `src/pages/_nav.html` | Shared top-nav partial (no wallet state) |

## Behaviour

- **404** — served for all unmatched routes; includes a friendly emoji illustration, copy, and "Back to Dashboard" CTA.
- **Error page** — displays an error code (defaults to 500) and a short message. Retry button calls `location.reload()`. Dashboard link always works. Error code and message can be overridden server-side or via `window.ErrorPage.set(code, message)` from an error boundary.
- Both pages include the identical top nav and require **no wallet / auth state**.

## Acceptance Criteria

- [x] 404 renders for all unmatched routes
- [x] Error page renders when error boundary catches an unhandled error
- [x] Both pages link back to a working route (`/dashboard`)
- [x] Pages are server-renderable (no wallet state dependency)
- [x] Both pages include the top nav
